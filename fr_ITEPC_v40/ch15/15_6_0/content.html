15.6 Description de l’installation, de la configuration et de la gestion d’un serveur de messagerie simple  

Un système de messagerie électronique utilise un logiciel client sur les périphériques des utilisateurs et un logiciel serveur sur un ou plusieurs serveurs de messagerie. Les clients lisent les courriels provenant du serveur de messagerie à l’aide de l’un des protocoles suivants :

  • protocole POP (Post Office Protocol) ;
  • protocole IMAP (Internet Message Access Protocol).

Les clients envoient leurs courriels à un serveur de messagerie et les serveurs de messagerie se transfèrent les courriels entre eux, à l’aide du protocole SMTP (Simple Mail Transfer Protocol).

Vous devez savoir comment configurer un ordinateur client pour qu’il accepte le format de courriel entrant approprié. Vous devez également comprendre le processus de configuration du serveur de messagerie. La configuration du logiciel de messagerie client peut être effectuée à l’aide d’assistants de connexion, comme indiqué dans la Figure 1. Les avantages et les inconvénients de chaque protocole de messagerie sont présentés dans la Figure 2.

SMTP
Le protocole SMTP (Simple Mail Transfer Protocol) envoie un courriel du client au serveur de messagerie, ou d’un serveur de messagerie à un autre. Le protocole SMTP possède les caractéristiques suivantes :

  • il s’agit d’un protocole textuel simple ;
  • il est envoyé par TCP via le port 25 ;
  • il est obligatoire pour tout envoi de courriel ;
  • le message est envoyé uniquement lorsque les destinataires sont identifiés et vérifiés.

POP
Le protocole POP (Post Office Protocol) est utilisé par le client de messagerie pour télécharger les courriels sur le serveur. La version la plus récente du protocole POP est POP3. Le protocole POP3 utilise généralement le port 110.

Il prend en charge les utilisateurs finaux disposant d’une connexion intermittente, par exemple une ligne commutée. Un utilisateur POP3 peut se connecter, télécharger un courriel sur le serveur, supprimer ce courriel, puis se déconnecter.

IMAP
Le protocole IMAP (Internet Message Access Protocol) est similaire au protocole POP3, mais offre des fonctionnalités supplémentaires. Tout comme le protocole POP3, il vous permet de télécharger un courriel sur le serveur à l’aide du client de messagerie. La différence est que le protocole IMAP permet à l’utilisateur d’organiser ses courriels sur le serveur de messagerie du réseau. Il est plus rapide que le protocole POP3 et exige moins d’espace disque sur le serveur ; toutefois, il utilise plus de ressources en termes d’UC. La version la plus récente du protocole IMAP est IMAP4. IMAP4 est souvent utilisé dans les réseaux de grande taille, par exemple sur les campus universitaires. Le protocole IMAP utilise généralement le port 143.

Serveur de messagerie électronique
Un serveur de messagerie est un ordinateur qui peut envoyer et recevoir des courriels de la part des clients de messagerie. Voici quelques serveurs de messagerie courants :

  • Microsoft Exchange ;
  • Sendmail ;
  • Eudora Internet Mail Server (EIMS).

Comme indiqué dans la Figure 3, des assistants et d’autres outils sont souvent mis à votre disposition pour vous guider dans la configuration du serveur de messagerie. Pour installer et configurer un serveur de messagerie, tel que Microsoft Exchange, vous devez tout d’abord vous assurer que le réseau dispose des qualifications requises et a été correctement configuré. Les serveurs Active Directory, les serveurs de catalogue global et les serveurs de nom de domaine (DNS) doivent être installés et en fonctionnement pour que Exchange puisse également être installé et fonctionner correctement. Un serveur Active Directory est un ordinateur qui héberge une base de données permettant une administration centralisée sur l’ensemble du réseau d’entreprise. Un serveur de catalogue global est un référentiel centralisé qui contient des informations sur chaque domaine du réseau d’entreprise.

Exchange doit être installé dans un domaine où chaque ordinateur fonctionne sous Windows 2000, ou une version ultérieure de Windows. C’est ce que l’on appelle le mode natif. Les contrôleurs de domaine Windows NT ne peuvent pas fonctionner dans un environnement natif.

La base de données Active Directory est organisée selon un schéma. Un serveur fonctionnant sous Windows 2003 est désigné comme contrôleur de schéma. Seul ce serveur pourra modifier l’organisation de la base de données utilisateur Active Directory. Lorsque l’administrateur réseau a besoin de modifier la structure Active Directory, cette modification est effectuée sur le contrôleur de schéma. Active Directory copie ensuite automatiquement la mise à jour sur les autres serveurs d’authentification.

Installation du serveur de messagerie
Il est préférable de tester l’environnement avant d’installer Exchange. Pour éviter que l’installation n’affecte le fonctionnement quotidien de votre réseau, configurez les services requis et installez Exchange sur un groupe dédié de serveurs, séparés du réseau principal. Séparez l’installation d’Exchange de votre réseau de production tant que vous n’êtes pas sûr qu’elle fonctionne correctement.

Avant d’installer Exchange, préparez les éléments suivants :

  • un déploiement DNS fiable et entièrement fonctionnel ;
  • un domaine Active Directory ;
  • un catalogue global au minimum ;
  • une fonctionnalité de domaine native Windows 2000 ou version supérieure ;
  • un logiciel serveur Exchange ;
  • des outils de support serveur Windows ;
  • un serveur contrôleur de schéma ;
  • une connexion Internet haut débit.

Vous êtes prêt à installer le serveur de messagerie lorsque toutes les qualifications de votre réseau sont en place. Vous devrez cependant ajouter IIS (Internet Information Services) à l’aide de l’assistant Ajout/Suppression de composants Windows, avant de commencer l’installation du serveur Exchange. IIS est un serveur doté de programmes servant à créer et à gérer des services de site Web. Vous pouvez lancer l’installation d’Exchange une fois IIS installé. Insérez le CD d’installation et lancez l’assistant Nouvelle installation Exchange.

L’assistant vous guide tout au long d’une procédure vous permettant de déterminer si Exchange est prêt à être installé. Il vérifie qu’IIS est installé, que les serveurs de domaine fonctionnent correctement et que les outils de support Windows sont installés. Le programme d’installation vous avertit en cas de problème au niveau de l’environnement d’installation. Dans ce cas, vous devez résoudre le problème, puis relancer le programme d’installation.

Une fois Exchange installé, le plug-in Microsoft Management Console pour Exchange, présenté dans la Figure 4, vous permet d’accéder aux différents paramètres de configuration à partir d’un emplacement pratique. Pensez à bien installer toutes les mises à jour pour que le serveur fonctionne correctement. Vous pouvez utiliser Exchange System Manager, une console qui contrôle le déploiement d’Exchange, pour gérer les options du serveur.

Servez-vous de la console ADUC (Active Directory Users and Computer) pour configurer la boîte aux lettres d’un utilisateur. Cette opération est également appelée « activation de la boîte aux lettres ».

Ouvrez la console ADUC pour créer un nouvel utilisateur. Complétez les champs de nom d’utilisateur et de mot de passe conformément à la stratégie de sécurité de domaine définie, comme indiqué dans la Figure 5. La boîte aux lettres de l’utilisateur est créée par le serveur Exchange lorsque l’utilisateur reçoit son premier courriel.

La configuration d’Exchange exige une planification minutieuse, impliquant notamment de vérifier que les serveurs, les services et les technologies réseau sont en place et fonctionnent correctement. Dans certains cas, une panne peut se produire durant l’installation ; il est alors parfois nécessaire de réinstaller le système d’exploitation et de recommencer la procédure d’installation d’Exchange.

REMARQUE : avant de planifier l’installation d’un serveur de messagerie, consultez des professionnels de la mise en réseau, des experts en réseau Windows ou des techniciens expérimentés spécialisés dans la messagerie.
