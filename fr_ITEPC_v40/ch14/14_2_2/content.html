14.2 Installation et configuration en local d’une imprimante et d’un scanner  
  14.2.2 Installation et configuration du pilote et du logiciel  

Les pilotes d’imprimantes sont des logiciels qui permettent aux ordinateur et aux imprimantes de communiquer les uns avec les autres. Le logiciel de configuration propose une interface qui permet aux utilisateurs de définir et de modifier les options de l’imprimante. Chaque modèle d’imprimante a son propre type de logiciel de configuration du pilote et du logiciel.

Lorsque vous connectez une nouvelle imprimante à un ordinateur, Windows XP tente de localiser et d’installer un pilote par défaut en utilisant l’utilitaire Plug and Play (PnP). Si Windows ne parvient pas à trouver le pilote nécessaire sur l’ordinateur, il tente de se connecter à Internet pour le trouver. Les fabricants d’imprimantes mettent fréquemment à jour les pilotes pour augmenter les performances de l’imprimante, ajouter de nouvelles options plus efficaces et résoudre les problèmes de compatibilité générale.

Installation du pilote de l’imprimante
Le processus d’installation et de mise à jour d’un pilote d’imprimante implique généralement les cinq étapes suivantes :

  1. Déterminez la version actuelle du pilote d’imprimante installé. N’oubliez pas de sélectionner une version plus récente pour améliorer les fonctionnalités.
  2. Cherchez, sur Internet, la version la plus récente du pilote.
  3. Téléchargez le pilote en suivant les instructions du site Web.
  4. Installez le pilote. La plupart des programmes d’installation de pilotes, une fois activés, installent automatiquement le nouveau pilote.
  5. Testez le pilote. Pour ce faire, sélectionnez Démarrer > Paramètres > Imprimantes et télécopieurs. Cliquez avec le bouton droit de la souris sur l’imprimante, puis choisissez Propriétés. Choisissez ensuite Imprimer une page de test. Si l’imprimante ne fonctionne pas, redémarrez l’ordinateur et réessayez.

La page de test imprimée doit contenir du texte lisible. Si ce n’est pas le cas, le problème peut provenir d’un pilote inadapté ou de l’utilisation du mauvais langage de description de page.

Langage de description de page (PDL)
Un langage de description de page (PDL) est un type de code qui décrit l’aspect d’un document dans un langage compréhensible pour une imprimante. Le PDL pour une page inclut le texte, les illustrations et les informations de formatage. Les logiciels utilisent les PDL pour envoyer à l’imprimante des images de ce que vous obtenez (WYSIWYG : à traduire littéralement par « ce que vous voyez est ce que vous obtenez »). L’imprimante traduit le fichier PDL pour que tout ce qui s’affiche à l’écran de l’ordinateur soit imprimé. Les PDL accélèrent le processus d’impression en envoyant de grandes quantités de données en une seule fois. Ils gèrent également les polices de l’ordinateur.

Il existe trois PDL courants :

  • Langage de commande pour imprimantes (PCL) : Hewlett-Packard a développé un langage PCL pour communiquer avec les premières imprimantes à jet d’encre. Le langage PCL est désormais une norme sectorielle pour la quasi-totalité des types d’imprimantes.
  • PostScript (PS) : Adobe Systems a développé le langage PS pour que les polices ou les types de texte partagent les mêmes caractéristiques à l’écran et sur le papier.
  • Graphics Device Interface (GDI) : l’interface GDI est un composant Windows permettant de gérer la manière dont les images graphiques sont transmises aux périphériques de sortie. L’interface GDI convertit les images dans un format bitmap utilisant l’ordinateur (et non l’imprimante) pour le transfert des images.
