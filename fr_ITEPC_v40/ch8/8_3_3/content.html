8.3 Description des principaux concepts et technologies de mise en réseau  
  8.3.3 Définition du protocole DHCP  

Le protocole DHCP (Dynamic Host Configuration Protocol) est un utilitaire logiciel utilisé pour attribuer des adresses IP de manière dynamique aux périphériques réseau. Ce processus dynamique évite d’avoir à affecter manuellement les adresses IP. Il est possible d’installer un serveur DHCP, qui se chargera de configurer les hôtes du réseau en leur attribuant automatiquement une adresse IP. Lorsqu’un ordinateur reçoit une adresse IP automatiquement, toutes les autres options d’adressage IP sont désactivées (cases grisées dans la Figure 1). Le serveur conserve une liste des adresses IP à attribuer et gère le processus d’adressage, de manière à ce que chaque système du réseau reçoive une adresse IP unique. Les adresses sont attribuées pour une période prédéterminée. Lorsque ce délai arrive à son terme, le serveur DHCP peut réutiliser l’adresse pour tout nouvel ordinateur ajouté au réseau.

Les informations suivantes sont associées aux hôtes par le serveur DHCP avec l’adresse IP :

  • adresse IP ;
  • masque de sous-réseau ;
  • passerelle par défaut ;
  • valeurs facultatives, telles que l’adresse du serveur DNS (système de noms de domaine).

Le serveur DHCP reçoit une requête en provenance de l’hôte. Il sélectionne l’adresse IP requise dans une liste d’adresses prédéfinies, stockée dans une base de données. Une fois l’adresse IP et les informations connexes sélectionnées, le serveur DHCP envoie ces valeurs à l’hôte ayant soumis la requête. Si l’hôte accepte cette offre, le serveur DHCP lui « loue » l’adresse IP pendant un laps de temps prédéfini.

L’utilisation d’un serveur DHCP simplifie l’administration du réseau, car le logiciel assure un suivi des adresses IP. La configuration TCP/IP automatique réduit également les risques d’adresses IP invalides ou dupliquées. Pour qu’un ordinateur en réseau puisse bénéficier des services du serveur DHCP, il doit d’abord identifier ce serveur sur le réseau local. Un ordinateur peut être configuré de manière à accepter une adresse IP proposée par un serveur DHCP ; pour ce faire, il suffit de cliquer sur l’option « Obtenir automatiquement une adresse IP », dans la fenêtre de configuration de la carte réseau, comme illustré dans la Figure 2.

Si votre ordinateur n’arrive pas à communiquer avec le serveur DHCP pour obtenir une adresse IP, le système d’exploitation Windows lui attribuera automatiquement une adresse IP privée. Si votre ordinateur reçoit une adresse IP située dans la plage 169.254.0.0 à 169.254.255.255, cela signifie qu’il ne pourra communiquer qu’avec des ordinateurs situés dans cette même plage. Une adresse privée peut parfois être utile, par exemple dans une salle de classe, lorsque vous souhaitez empêcher tout accès extérieur au réseau. Cette fonction du système d’exploitation est appelée APIPA (Automatic Private IP Addressing, adressage IP privé automatique). La fonction APIPA demandera continuellement au serveur DHCP une adresse IP pour votre ordinateur.
