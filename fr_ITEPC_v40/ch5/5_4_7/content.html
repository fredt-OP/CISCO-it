5.4 Installation d’un système d’exploitation  
  5.4.7 Identification des fichiers de séquence d’amorçage et de registre  

Vous devez connaître le processus utilisé par Windows XP lors de l’amorçage. La compréhension de ces étapes peut vous aider par la suite à résoudre des problèmes d’amorçage. La Figure 1 présente la séquence d’amorçage de Windows XP.

Processus d’amorçage de Windows XP
Pour démarrer le processus d’amorçage, mettre d’abord l’ordinateur sous tension. Cela s’appelle un démarrage à froid. L’ordinateur effectue le test automatique de mise sous tension (POST). Comme l’adaptateur vidéo n’a pas encore été initialisé, toutes les erreurs se produisant à ce moment dans le processus d’amorçage se traduisent par une série de signaux sonores, appelés codes de bips.

Après le POST, le BIOS localise et lit les paramètres de configuration enregistrés dans le CMOS. Ce paramètre de configuration indique l’ordre dans lequel les périphériques sont testés pour détecter si un système d’exploitation s’y trouve. Le BIOS amorce l’ordinateur à l’aide du premier disque contenant un système d’exploitation.

Une fois le disque contenant le système d’exploitation trouvé, le BIOS localise l’enregistrement d’amorçage maître (MBR). Le MBR localise le chargeur d’amorçage. Sous Windows XP, le chargeur d’amorçage est appelé chargeur NT (NTLDR).

NTLDR et le menu d’amorçage Windows
À ce stade, NTLDR contrôle plusieurs étapes d’installation. Par exemple, si le disque contient plusieurs systèmes d’exploitation, BOOT.INI permet à l’utilisateur de sélectionner le système à utiliser. En cas d’absence de système d’exploitation ou si l’utilisateur n’en sélectionne aucun avant l’expiration du compteur, les actions suivantes se produisent :

  • NTLDR exécute NTDETECT.COM pour obtenir des informations sur le matériel installé.
  • NTLDR utilise ensuite le chemin indiqué dans le fichier BOOT.INI pour trouver la partition d’amorçage.
  • NTLDR charge deux fichiers formant le noyau de XP : NTOSKRNL.EXE et HAL.DLL.
  • NTLDR lit les fichiers de registre, choisit un profil de matériel et charge les pilotes des périphériques.

Registre Windows
Les fichiers de registre Windows jouent un rôle important dans le processus d’amorçage de Windows XP. Ces fichiers se reconnaissent à leur nom bien distinct, qui commence par HKEY_, comme indiqué dans la Figure 2, suivi du nom de la partie du système d’exploitation qu’ils contrôlent. Chaque paramètre de Windows, depuis l’arrière-plan du bureau et la couleur des boutons écran jusqu’aux licences octroyées pour les applications, est enregistré dans le registre. Lorsqu’un utilisateur change les paramètres du Panneau de configuration, les associations de fichiers, les stratégies de sécurité système ou les logiciels installés, les modifications sont enregistrées dans le registre.

Chaque utilisateur a une section unique dans le registre. Le processus d’ouverture de session Windows utilise les paramètres système se trouvant dans le registre pour reconfigurer le système dans l’état où il était la dernière fois que l’utilisateur l’a mis sous tension.

Noyau NT
À ce stade, le noyau NT, qui constitue le cœur du système d’exploitation Windows, prend le relais. Ce fichier s’appelle NTOSKRNL.EXE. Il démarre le fichier d’ouverture de session appelé WINLOGON.EXE et affiche l’écran d’accueil XP.

REMARQUE : si un disque SCSI amorce l’ordinateur, Windows copie le fichier NTBOOTDD.SYS pendant l’installation. Si les disques SCSI ne sont pas utilisés, ce fichier n’est pas copié.
