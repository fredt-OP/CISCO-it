5.4 Installation d’un système d’exploitation  
  5.4.9 Description des structures de répertoires  

Extensions et attributs de fichier
Sous Windows, les fichiers sont organisés dans une structure de répertoires. Le niveau racine de la partition Windows est généralement appelé lecteur C:\. On trouve ensuite un jeu initial de répertoires normalisés, appelés dossiers, pour le système d’exploitation, les applications, les informations de configuration et les fichiers de données. Après l’installation initiale, les utilisateurs peuvent installer la plupart des applications et données dans le répertoire de leur choix.

Les fichiers contenus dans une structure de répertoires respectent une convention d’attribution de noms Windows :

  • 255 caractères au maximum peuvent être utilisés ;
  • des caractères tels que le point (.) ou la barre oblique (\ /) ne sont pas acceptés ;
  • une extension de trois ou quatre lettres est ajoutée au nom du fichier pour identifier le type de fichier ;
  • les noms de fichiers ne sont pas sensibles à la casse.

Les extensions de fichiers suivantes sont couramment utilisées :

  • .doc : Microsoft Word ;
  • .txt : texte ASCII uniquement ;
  • .jpg : format graphique ;
  • .ppt : Microsoft PowerPoint ;
  • .zip : format de compression.

La structure de répertoires conserve un jeu d’attributs pour chaque fichier, qui contrôle la manière dont le fichier peut être affiché ou modifié. Les attributs de fichier les plus répandus sont les suivants :

  • R : le fichier est en lecture seule.
  • A : le fichier sera archivé la prochaine fois que le disque sera sauvegardé.
  • S : le fichier est marqué comme étant un fichier système et un avertissement est donné en cas de tentative de suppression ou de modification du fichier.
  • H : le fichier est caché dans l’affichage des répertoires.

Vous pouvez visualiser les noms de fichiers, les extensions et les attributs en entrant dans une fenêtre DOS et en utilisant la commande ATTRIB, comme indiqué dans la Figure 1. Utilisez le chemin suivant :

Démarrer > Exécuter > cmd

Accédez au dossier contenant le fichier qui vous intéresse. Tapez ATTRIB, suivi du nom de fichier. Utilisez un caractère générique tel que *.* pour afficher plusieurs fichiers en même temps. Les attributs de chaque fichier apparaissent à l’écran dans la colonne de gauche. Vous pouvez obtenir des informations sur la commande ATTRIB en tapant, à l’invite de commande :

ATTRIB/?

Il est possible d’accéder à l’équivalent Windows de la commande ATTRIB en cliquant avec le bouton droit de la souris sur un fichier dans l’Explorateur Windows et en sélectionnant Propriétés.

REMARQUE : pour afficher les propriétés d’un fichier dans l’Explorateur Windows, vous devez d’abord régler celui-ci sur « Afficher les fichiers cachés ». Pour ce faire, utilisez ce chemin :

Cliquez avec le bouton droit de la souris sur Démarrer > Explorer > Outils > Options des dossiers > Affichage

NTFS et FAT32
Windows XP et Windows 2000 utilisent les systèmes de fichiers FAT32 et NTFS. L’une des principales différences entre ces deux systèmes de fichiers concerne la sécurité. NTFS peut prendre en charge plus de fichiers et des fichiers plus volumineux que FAT32. Il offre également des fonctions de sécurité plus flexibles pour les fichiers et dossiers. Les Figures 2 et 3 montrent les propriétés d’autorisation de fichiers avec FAT32 et NTFS.

Les partitions peuvent être converties de FAT32 à NTFS à l’aide de l’utilitaire CONVERT.EXE. Cela permet de bénéficier des avantages de sécurité supplémentaires qu’offre NTFS. Pour restaurer une partition NTFS et revenir à une partition FAT32, reformatez la partition et restaurez les données à partir d’une sauvegarde.

ATTENTION : avant de convertir un système de fichiers, n’oubliez pas de sauvegarder les données.
