16.2 Sélection de composants de sécurité selon les besoins d’un client  
  16.2.1 Description et comparaison des techniques de sécurité  

Un technicien doit déterminer les techniques appropriées pour sécuriser l’équipement et les données de son client. Selon la situation, plusieurs techniques peuvent être requises.

Mots de passe
L’utilisation d’informations de connexion sécurisées et chiffrées pour les ordinateurs auxquels on accède via un réseau doit être une exigence minimale dans toute organisation. En effet, des logiciels malveillants surveillent le réseau et peuvent conserver des mots de passe en texte brut. Si les mots de passe sont chiffrés, les pirates doivent décoder le chiffre avant de découvrir les mots de passe.

Journalisation et audit
La journalisation et l’audit des événements doivent être activés afin de surveiller l’activité du réseau. L’administrateur réseau audite le fichier journal des événements pour examiner les accès réseau par des utilisateurs non autorisés.

Configuration sans fil
Les connexions sans fil sont particulièrement vulnérables aux accès par des pirates. Les clients sans fil doivent être configurés pour chiffrer les données.

Chiffrement
Les technologies de chiffrement sont utilisées pour coder les données transmises sur un réseau. Chaque technologie est utilisée dans un but précis :

  • Codage par hachage : le hachage assure que les messages ne sont pas corrompus ou manipulés au cours de leur transmission. Cette technique utilise une fonction mathématique pour créer une valeur numérique unique pour les données. Même si un seul caractère est modifié, le résultat de la fonction, appelé algorithme MD, diffère. Cependant, ce codage ne fonctionne que dans un seul sens. Le fait de connaître l’algorithme MD ne permet pas à un pirate de recréer le message. Il devient donc difficile d’intercepter les messages et de les modifier. Le codage par hachage est illustré à la Figure 1. Les algorithmes de hachage les plus utilisés sont SHA et MD5.
  • Chiffrement symétrique : ce chiffrement exige que les deux interlocuteurs d’une conversation chiffrée utilisent une clé de chiffrement capable de coder et de décoder les données. L’expéditeur et le récepteur doivent utiliser des clés identiques. Le chiffrement symétrique est illustré à la Figure 2.
  • Chiffrement asymétrique : le chiffrement asymétrique exige deux clés, l’une privée et l’autre publique. Une clé privée est requise pour écrire un message et une clé publique est utilisée pour décoder le message. L’avantage du chiffrement asymétrique est le suivant : seule la clé privée doit être maintenue secrète. Les clés publiques peuvent être distribuées ouvertement par courriel ou publiées sur le Web. Le chiffrement asymétrique est illustré à la Figure 3.
  • Réseau privé virtuel : un réseau privé virtuel utilise le chiffrement pour sécuriser les données comme si elles étaient transférées sur un réseau local privé d’entreprise, même si elles circulent en réalité sur un réseau quelconque, par exemple Internet. Les pipelines de données sécurisés entre les points du réseau privé virtuel sont appelés des « tunnels sécurisés ». Ce processus est illustré à la Figure 4.
