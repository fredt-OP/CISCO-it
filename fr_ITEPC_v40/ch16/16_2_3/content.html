16.2 Sélection de composants de sécurité selon les besoins d’un client  
  16.2.3 Description et comparaison des types de pare-feu  

Les pare-feux matériels et logiciels protègent les données et l’équipement d’un réseau contre les accès non autorisés. Un pare-feu doit être utilisé en plus des logiciels de sécurité.

Les pare-feux matériels et logiciels disposent de plusieurs modes de filtrage du trafic des données du réseau :

  • Filtre de paquet : ensemble de règles autorisant ou interdisant le trafic sur la base de critères tels que les adresses IP, les protocoles ou les ports utilisés.
  • Pare-feu proxy : pare-feu vérifiant tout le trafic et autorisant ou interdisant les paquets selon des règles configurées. Un proxy fonctionne comme une passerelle qui protège les ordinateurs à l’intérieur du réseau.
  • Inspection dynamique de paquets : pare-feu conservant une trace de l’état des connexions réseau qui le traversent. Les paquets ne faisant pas partie d’une connexion connue ne sont pas autorisés à traverser à nouveau le pare-feu.

Pare-feu matériel
Un pare-feu matériel est un composant de filtrage physique qui inspecte les paquets de données provenant du réseau avant que ceux-ci n’atteignent les ordinateurs et les autres périphériques du réseau. Les pare-feux matériels sont souvent installés sur des routeurs. Un pare-feu matériel constitue une unité autonome qui n’utilise pas les ressources des ordinateurs qu’il protège ; il n’en affecte donc pas les performances.

Pare-feu logiciel
Un pare-feu logiciel est une application s’exécutant sur un ordinateur pour inspecter et filtrer les paquets de données. Le pare-feu Windows est un exemple de pare-feu logiciel qui fait partie du système d’exploitation Windows. Un pare-feu logiciel utilise les ressources de l’ordinateur, si bien que, pour l’utilisateur, les performances de l’ordinateur sont réduites.
