12.2 Installation, configuration et optimisation d’un système d’exploitation  
  12.2.2 Installation de Windows XP Professionnel en utilisant une installation personnalisée  

L’installation par défaut de Windows XP Professionnel est suffisante pour la plupart des ordinateurs utilisés dans un environnement domestique ou d’une petite entreprise. Une installation personnalisée de Windows XP Professionnel peut permettre de gagner du temps et de fournir une configuration homogène du système d’exploitation sur les ordinateurs d’un réseau de grandes dimensions.

  • Installation sans surveillance depuis un point de distribution réseau, à l’aide d’un fichier de réponses.
  • Installation basée sur image, au moyen de Sysprep et d’un programme d’image de disque, qui copie une image du système d’exploitation directement sur le disque dur sans intervention de l’utilisateur.
  • Installation à distance à l’aide des services RIS (Remote Installation Services), qui permettent de télécharger l’installation via le réseau. Cette installation peut être demandée par l’utilisateur ou forcée par l’administrateur pour un poste spécifique.
  • OS Deployment Feature Pack utilisant Microsoft Systems Management Server (SMS), qui peut simplifier considérablement le déploiement d’un système d’exploitation dans toute une organisation.

Installation sans surveillance
L’installation sans surveillance utilisant le fichier de réponses « unattend.txt » constitue la méthode d’installation personnalisée la plus facile à effectuer sur un réseau. Un fichier de réponses peut être créé à l’aide d’une application appelée setupmgr.exe, située dans le fichier deploy.cab sur le CD de Windows XP Professionnel.

La Figure 1 présente un exemple de fichier de réponses. Une fois que vous avez répondu à toutes les questions, le fichier unattend.txt est copié dans le dossier partagé de distribution, sur un serveur. À ce stade, deux options sont possibles :

  • Exécution du fichier unattended.bat sur la machine client. Cela prépare le disque dur et installe automatiquement le système d’exploitation depuis le serveur, via le réseau.
  • Création d’un disque d’amorçage, qui amorce l’ordinateur et le connecte au partage de distribution sur le serveur. Exécutez le fichier batch pour installer le système d’exploitation sur le réseau.

Installation basée sur image
En cas d’installations basées sur image, vous devez tout d’abord configurer entièrement l’un des ordinateurs pour qu’il soit opérationnel. Exécutez ensuite Sysprep, afin de préparer le système pour l’image du disque. Une application tierce d’image du disque prépare une image de l’ordinateur configuré, qui peut ensuite être gravée sur un CD ou un DVD. Cette image peut ensuite être copiée sur des ordinateurs de couche HAL compatible, afin de réaliser l’installation sur plusieurs ordinateurs. Une fois l’image copiée, vous pouvez amorcer l’ordinateur, mais il peut être nécessaire de configurer certains paramètres, tels que le nom de l’ordinateur et l’appartenance au domaine.

Installation à distance
L’utilisation des services d’installation à distance RIS ressemble beaucoup à une installation basée sur image, sauf qu’il est inutile d’utiliser un utilitaire d’image du disque. Vous pouvez utiliser les services RIS pour configurer à distance de nouveaux ordinateurs Microsoft Windows, en vous servant d’un dossier partagé de réseau RIS comme source des fichiers du système d’exploitation Windows. Vous pouvez installer des systèmes d’exploitation sur des ordinateurs clients activés pour un amorçage à distance. Les ordinateurs clients connectés au réseau peuvent être démarrés en utilisant un adaptateur de réseau PXE (Pre-Boot eXecution Environment) ou un disque d’amorçage à distance. Le client ouvre ensuite une session en utilisant des identifiants de compte utilisateur valides.

Les services RIS sont conçus pour être utilisés dans un réseau de dimensions relativement réduites et ne doivent pas être utilisés sur les liaisons à bas débit d’un réseau étendu. Microsoft System Management Server (SMS) permet à un administrateur réseau de gérer un grand nombre d’ordinateurs sur un réseau. SMS peut être utilisé pour gérer des mises à jour, fournir un contrôle à distance et effectuer une gestion d’inventaire. En option, une fonction permet le déploiement d’un système d’exploitation, ce qui requiert l’installation du pack OS Deployment Feature Pack SMS sur le serveur Windows 2003. SMS permet l’installation d’un grand nombre d’ordinateurs clients sur l’intégralité d’un réseau, local ou étendu.
